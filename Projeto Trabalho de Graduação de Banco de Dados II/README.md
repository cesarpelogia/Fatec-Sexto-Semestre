### 4º Semestre - Sistema WEB de Dados Espaciais

#### **Informações**:
- **Data:** *2025 - 1º Semestre*
- **Empresa:** *Visiona*

#### **Proposta:** 
<p align="justify">
O projeto teve como objetivo desenvolver um sistema web para edição e análise de dados geoespaciais no contexto agrícola.

A solução permite manipulação de polígonos, análise de dados e acompanhamento por meio de dashboards, contribuindo para a melhoria de modelos de inteligência artificial.
</p>

**GitHub:** [Demeter](https://github.com/Morpheus-Fatec/API_4S_Visiona_PolygonEditor)

#### **Tecnologias Utilizadas**

Java • JavaScript • PostgreSQL • PostGis • SpringBoot • Vue.Js • Leaflet • Pinia • Git •  • Github 

#### **Contribuições Pessoais**

O quarto semestre manteve a formação da equipe, porém com a substituição do Product Owner, o que alterou significativamente a dinâmica de trabalho. A nova abordagem, mais direta e exigente, gerou dificuldades de adaptação em parte do time, resultando em conflitos internos e queda de engajamento de alguns integrantes ao longo das sprints.

Paralelamente, o projeto apresentou um salto relevante de complexidade técnica, com a introdução de dados geoespaciais. Durante a primeira sprint, houve uma mudança crítica de escopo, pois os dados fornecidos pelo cliente — imagens de satélite de grande volume e geometrias com alta complexidade — foram disponibilizados apenas durante a execução, exigindo replanejamento completo das atividades.

Os principais desafios envolveram:

- Manipulação de imagens de grande porte, inviáveis para persistência direta em banco de dados
- Processamento e edição de grandes volumes de polígonos geoespaciais
- Necessidade de garantir integridade e performance nas operações espaciais

Como solução, adotamos uma arquitetura baseada em armazenamento externo de imagens (AWS) e utilização de banco geoespacial com PostGIS. Atuei diretamente na implementação do fluxo de conversão entre formatos, realizando a transformação bidirecional entre GeoJSON e estruturas geométricas (MultiPolygon via JTS), garantindo compatibilidade entre API, aplicação e banco de dados.

Além disso, também contribuí na implementação de consultas espaciais avançadas, incluindo operações de interseção para identificação de divergências entre classificações (false positives e false negatives), bem como na estruturação do retorno em formato GeoJSON para consumo pelo front-end.

No âmbito de gestão, o semestre exigiu maior rigor no acompanhamento das entregas, incluindo a adoção de geração automatizada de gráfico burndown e controle mais estrito de prazos. Também houve a necessidade de adaptação ao uso de repositórios com submódulos, ampliando a complexidade de versionamento.

Minhas contribuições foram consolidadas nas seguintes frentes principais:

1 - Processamento e Persistência de Dados Geoespaciais

- Implementei a conversão bidirecional entre GeoJSON e estruturas geométricas (MultiPolygon), garantindo integridade dos dados (ex.: fechamento de anéis e validação estrutural).
- Estruturei a persistência utilizando PostGIS com definição de SRID adequado (4326).
- Implementei o fluxo completo de dados: entrada via API → conversão → persistência → retorno em GeoJSON.

2 - Consultas Espaciais e Regras de Negócio

- Desenvolvi consultas espaciais utilizando ST_Intersects para identificação de inconsistências entre classificações.
- Estruturei endpoints para exposição desses dados (false positives / false negatives).
- Ajustei contratos de API e padronizei retorno via DTOs no formato GeoJSON.
- Implementei regras de controle de edição baseadas em responsabilidade (analista/consultor), evitando conflitos concorrentes.

3 - Arquitetura e Estabilidade da Aplicação

- Refatorei fluxos de retorno de dados, estruturando manualmente payloads para maior controle sobre a resposta da API.
- Corrigi problemas de estabilidade (ex.: inicialização de coleções para evitar NullPointerException).
- Atuei na organização de serviços e integração entre camadas.

4 - Documentação e Organização do Projeto

- Reestruturei o README, melhorando navegação, clareza e apresentação para stakeholders.
- Mantive cronograma e backlog alinhados com a execução real do projeto.
- Criei documentação detalhada de dashboards, fluxos operacionais e regras de negócio.
- Estruturei manuais com evidências visuais e organizei a navegação entre documentos.
- Garanti consistência da documentação via fluxo de PR e integração contínua.

#### **Hard Skills Efetivamente Desenvolvidas**

Neste semestre, houve um avanço significativo em temas técnicos mais especializados, com destaque para:

- Manipulação e modelagem de dados geoespaciais
- Uso de PostGIS para armazenamento e consulta de geometrias
- Conversão entre formatos (GeoJSON ↔ estruturas geométricas)
- Implementação de consultas espaciais (ex.: interseções)
